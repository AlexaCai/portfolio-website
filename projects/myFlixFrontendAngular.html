myFlix web app (frontend - Angular)

For details about the backend side, see this related project

A single page responsive web application for all movie fans
Overview

myFlix was created to serve as a reference in the domain of visual entertainment. Users can create an account and then log into myFlix to have access to information about different movies. They can search for movies based on different genres or directors, and create a list of favorite movies.

Purpose

Learn and consolidate my skills with Angular during my full-stack web development training. Develop and showcase an appealing and complete web app (client side and server side) built using TypeScript and Angular Material. This page concerns the frontend part - Angular version (see React version here).

Objective

Develop an easy-to-use and responsive web app for users to access whenever they want to read and save information about different movies, as well as allowing me to put into practice my skills regarding the interaction between backend and frontend for a fully functional web app.

Tech summary

myFlix is a single-page responsive web app (SPA), designed with state-based routing and several interface views. The design is responsive to be optimal on any device types and the UI has a polished visual for the best user experience possible. The app is powered on the backend by an API built by myself using Node.js and Express framework (see more details on my related backend API project here).

Tools used
  • Angular
  • Angular Material
  • TypeScript
  • HTML
  • CSS

Retrospective


The original objective of this project (along with this backend API related project) was to develop a complete web application (client-side and server-side) built using Angular and Angular Material, which align with the final result.

Being in my first steps with Angular, the most challenging thing for me was to understand the working logic of data interpolation and data-binding. It also took me a certain time to get familiar with Angular Material, since I had more experience with React Bootstrap up until this project. However, these two libraries still present some similarities, which helped to get used to Angular Material quickly.

Throughout this project, I particularly enjoyed learning about Angular in general and developing skills in this very popular frontend framework. It was interesting to see how Angular can achieve similar things as React, but in different ways (and vice versa). Using Angular Material was also pleasant since it provides a wide range of UI elements to facilitate professional and consistent UIs.

Coding this project with a clear workflow gave me the opportunity to acquire a good understanding of how Angular works both in theory and in practice.

In the end, my hope with this web app is that any user using it have a simple, pleasant and easy-to-use experience to help them in their quest for their next weekend movie marathon.
