Pokedex web app

A Pokedex web app for all Pokemon fans
Overview

The Pokedex web app has been created to serve as an information resource to any aspiring Pokemon Masters. Users can open the Pokedex and see a list of Pokemons displayed. They can then click on any Pokemon to show more information on the selected Pokemon.

Purpose

Consolidate my skills on working with external APIs and manipulate their data to render it in an aesthetically, pleasing and easy-to-use way, as well as working on the responsiveness aspect of a web app to make it fit to use on any devices - small or large.

Objective

Build a web app with HTML, CSS, JavaScript and jQuery that loads data from an external API (using AJAX) to enable the viewing of data points in detail and based on user interaction, as well as using some Bootstrap features for the UIs.

Tech summary

The web app is built with HTML, CSS, JavaScript, jQuery and Bootstrap, and is fed by an external API named PokeAPI. PokeAPI is a RESTful API linked to a database detailing everything about the Pokemons. The API documentation can be found here. All the web app code is deployed and available on Github.

Retrospective


The original objective of this project was to build a web app with HTML, CSS, JavaScript, jQuery and Bootstrap that loads data from an external API (using AJAX) to enable the viewing of data points in detail and based on user interaction, which align with the final result.

Being in my first steps in working with external API using AJAX, the most challenging thing for me was to understand the logic of promises for asynchronous code. More precisely, to understand the basic structure of a promise, how it behaves if it’s resolved or rejected, and the chaining concept with .then( ) and .catch( ).

This was however also the thing I found the most stimulating and interesting to learn about, as they provide a good alternative to the traditional callback for asynchronous codes. Beside this, working with Bootstrap to create appealing and professional responsive visuals is also something I enjoyed a lot, and for which I’ll try to consistently improve myself.

Coding this project gave me the opportunity to acquire a good understanding on how to work with external APIs and how to manipulate their data, both in theory and in practice. With the knowledge I acquired along the way, I would feel equipped to work with other types of APIs to create web app of other nature.


In the end, my hope with this web app is that any user using it has a simple, pleasant and easy-to-use experience to help them in their quest of becoming the next Pokemon Master.
