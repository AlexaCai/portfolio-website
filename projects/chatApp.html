Chat app

A React Native mobile Chat app allowing users to talk and share medias
Overview

Chat app was created to allow people to talk and exchange on a mobile app. Users can send messages like other very common chat apps (WhatsApp, Messenger, etc.). They can also send images, location and audios. Users can also customize their application by selecting the color background they want to use for the chat UI.

Purpose

Learn and consolidate my skills with the creation of React Native app during my full-stack web development training to develop a user-friendly, appealing and easy-to-use chatting mobile application, as well as working with Firestore (a NoSQL cloud-based database service offered by Google) to handle the chat app data.

Objective

Build a chat app for mobile devices using React Native and Google Firebase features to implement a database, storage space and anonymous authentication process. Coding the app to ensure users can send images, location and audios on top of messages, access their messages when offline and allow them customize their experience.

Tech summary

Chat app is built with React Native. The application Expo Go and Android Studio have been used for the development. The chat interface have been created using the Gifted Chat library. All chat messages are stored in a Google Firestore Database and locally on users’ devices. Images and audios sent on Chat app are also stored in a Google Cloud Storage created on Firebase, and the app authenticates users anonymously via Firebase Authentication. The app also allows users to consult their messages offline by using client-side storage (AsyncStorage pack. from React Native).

Meet app development process


As for any projects, having a clear workflow is important to build a product in a logical and methodological way, whether in a team or in a solo project. The following summarize how Meet app has been built, as well as the challenges faced and the solutions found along the way.

Retrospective


The original objective of this project was to build a serverless, progressive web application (PWA) with React using Google OAuth / AWS Lambda for users authentication and Google Calendar API to fetch different events, all of it under the Test Driven development (TDD) and Behavior Driven Development (BDD) techniques, which align with the final result.

Being in my first steps with many topics covered in this project (OAuth, serverless functions, AWS Lambda, TDD, BDD, PWA and visualization libraries), the most challenging thing for me was to understand the working logic of all these new tools / concepts, especially regarding authentication flow with Google OAuth and AWS Lambda. I add to do a lot of extra reading, cross researches and real-life case study to better understand how it works.

While complex at first, I still enjoyed learning more about OAuth process and serverless functions, as we can see those types of logic being used in the real world by many other applications.


I also enjoyed a lot learning about the different testing techniques (TDD and BDD) since they put forward a specific and practical development process that has many advantages, while ensuring that the codes written for an app different features always work along the way, step by step (and therefore allows me to quickly spot and correct any anomalies along the way as well).


As for the PWA precisely, it was very interesting to learn how an app can be converted to be installable on a desktop or a mobile phone, as well as working offline. These two characteristics can greatly enhance user-experience, making PWAs very interesting in web development.


Finally, regarding the visualization tools I’ve used at the end of this project, they are only a fraction of what is possible to represent visually with Recharts. The use of visuals from this library in this project therefore allowed me to understand quite well the functioning Recharts and opened the door to the possible use (for other projects for example) of other different forms of visual representations that could be extremely useful in many different contexts.

Fortunately, I already had a good base working with React (main technology used for Meet app), which still allows me, despite all the new tools, technologies and topics emerging from this project, to navigate into the development process with a good and solid understanding of some of its key features.
