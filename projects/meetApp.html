Meet web app

A serverless and progressive web app to locate coding events all around the world

Project brief

Overview

Meet app was created to serve as a resource for individuals to find different events. Users have the ability to explore coding events of various types in different cities. They can also customize the number of events they want to display on their screen. All event information is presented visually with a dynamic scatter plot and pie chart, and in a list below the visuals.

Purpose

Learn and consolidate my skills with the creation of serverless and progressive web application (PWA), test writing techniques and application performance monitoring (APM) to develop a user-friendly, appealing and easy-to-use web application, as well as working on data visualization techniques using a specific React library.

Objective

Build a serverless, progressive web application with React using Google OAuth / AWS Lambda for users authentication and Google Calendar API to fetch different events. The objective was also to develop this application under the Test-Driven Development (TDD) and Behavior-Driven Development (BDD) techniques.

Tech summary

Meet app is built with React, and designed with CSS. It has a serverless backend handled and hosted on AWS Lambda. To use the web app, users are required to enter their Google credentials since a OAuth2 authentication flow using Google OAuth has been implemented. The events displayed within Meet app are fetched from the Google Calendar API. The different tests have been carried out using a mix of Jest, Cucumber and Puppeteer, and the graphs have been created using Recharts (React visualization library). Finally, the web app performance has been monitored using Atatus (APM tool).

Development process

As for any projects, having a clear workflow is important to build a product in a logical and methodological way, whether in a team or in a solo project. The following summarize how Meet web app has been built, as well as the challenges faced and the solutions found along the way.

Retrospective


The original objective of this project was to build a serverless, progressive web application (PWA) with React using Google OAuth / AWS Lambda for users authentication and Google Calendar API to fetch different events, all of it under the Test Driven Development (TDD) and Behavior Driven Development (BDD) techniques, which align with the final result.

Being in my first steps with many topics covered in this project (OAuth, serverless functions, AWS Lambda, TDD, BDD, PWA and visualization libraries), the most challenging thing for me was to understand the working logic of all these new tools / concepts, especially regarding authentication flow with Google OAuth and AWS Lambda. I had to do a lot of extra reading, cross researches and real-life case study to better understand how it works.

While complex at first, I still enjoyed learning more about OAuth process and serverless functions, as we can see those types of logic being used in the real world by many other applications.


I also enjoyed a lot learning about the different testing techniques (TDD and BDD) since they put forward a specific and practical development process that has many advantages, while ensuring that the codes written for different features always work along the way, step by step (and therefore allows to quickly spot and correct any anomalies along the way as well).


As for the PWA precisely, it was very interesting to learn how a web app can be converted to be installable on a desktop or a mobile phone, as well as working offline. These two characteristics can greatly enhance user experience, making PWAs very interesting in web development.


Finally, regarding the visualization tools I’ve used at the end of this project, they are only a fraction of what's possible to do visually with Recharts. Still, the use of visuals from this library allowed me to understand well the functioning of Recharts and opened me the doors to use different forms of visual representations that could be very useful in different contexts / other projects.

I already had a good base working with React up to this point (main technology used for Meet web app), which allowed me, despite all the new tools, technologies and topics emerging from this project, to navigate into the development process with a good and solid understanding of some of it's key features.

Coding this project with a clear workflow also helped me to acquire a good understanding of the many new things encountered along the way, both in theory and in practice.

In the end, my hope with this web app is that any user using it has a simple, pleasant and easy-to-use experience to help them in their quest to find the next coding event they would like to attend, whether the event being on React, jQuery, or Node.js, and whether the event being in Dubai, Toronto or Tokyo.
