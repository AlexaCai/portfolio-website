myFlix web app (frontend - React)


For details about the backend side, see this related project

A responsive single page web application for all movie fans

Project brief

Overview

myFlix was created to serve as a reference in the domain of visual entertainment. Users can create an account and then log into myFlix to have access to information about different movies. They can search for movies, filter results based on different criteria and create a list of their favorite movies.

Purpose

Learn and consolidate my skills with frontend technologies, API interaction and layout design by developing a comprehensive & visually appealing web application using JavaScript and the MERN tech stack. This page concerns the frontend part made with React.

Objective

Develop an easy-to-use and responsive web app for users to access whenever they want to read and save information about different movies, as well as implementing different forms so that users can communicate with the API and perform different CRUD operations.

Tech summary

myFlix is a responsive single-page web app (SPA), designed with state-based routing and several interface views. The design is responsive to be optimal on any device types and the UI has a polished visual for the best user experience possible. The web app is supported on the backend by an API built by myself using Node.js and Express framework (see more details on the related backend API project here).

Retrospective


The original objective of this project (along with this backend API related project) was to develop a complete web app (client-side and server-side) built using JavaScript technologies and the MERN tech stack (MongoDB, Express, React, and Node.js), which align with the final result.

Being in my first steps with React, the most challenging thing for me was to understand the working logic of key concepts like parent-child component relationships, states, props and state-based routing.

These same elements were however also the ones I found the most stimulating and interesting to learn about, as they truly show all the power of a well designed React web app, and because I knew I could recycle that new knowledge and expand it in any other apps I would like to build in the future. Creating appealing and professional responsive visuals is also something I enjoyed a lot, and for which I’ll try to consistently improve myself.

Coding this project with a clear workflow gave me the opportunity to acquire a good understanding of how React works both in theory and in practice. With the knowledge I acquired along the way, I also thought about other elements that could possibly be added to myFlix and that I would feel equipped to implement eventually:

  • Presenting more varied information on each film
  • Increasing the filtering options for more choices
  • Allowing users to create different types of lists

    • In the end, my hope with this web app is that any user using it has a simple, pleasant and easy-to-use experience to help them in their quest for their next weekend movie marathon - popcorn or not.
