Recipe web app (Django version)

A complete Django web app to create, search and modify recipes for all food lovers

Project brief

Overview

This Recipe web app was created to serve as a central hub for international cuisine lovers. Users can go on the web app and access many recipes from all around the globe, search for recipes by name or via advanced search, and even submit their own recipe. Users can also create an account for a more complete experience, such as being able to create their own private recipes, add recipes to their favorite and comment recipes publicly.

Purpose

Learn and consolidate my skills with Django web framework both on the backend and frontend, while creating a fully working Recipe web application. More precisely, the purpose was to improve my abilities in implementing a project structure in the Django way (apps, urls.py, views.py, etc), while using at the same time my skills with other tools such as JavaScript and Bootstrap to add life to the web app on the frontend side.

Objective

Build a Recipe web app using Django and a SQLite database to provide an easy-to-use interface for all food lovers, supported by simple forms and concise instructions that any user can follow. Some sub-objectives were to 1) allow both signed and unsigned users to use the web app, 2) build a personalized Django admin interface adapted to the needs for more efficient backend work, and 3) develop a clear frontend visual for the best possible user experience.

Tech summary

This Recipe web app is built with Django version 4.2.6, Python version 3.8.7 and an SQLite database. The frontend is built with HTML, Bootstrap, CSS and JavaScript. The web app is deployed on Heroku. Even if not part of the final product, QuerySet API, DataFrames (pandas) and plotting libraries (matplotlib - a comprehensive library for creating static, animated, and interactive visualizations in Python) have been tried out during development.

Tools used
  • Django & Python
