myFlix web app (backend)

For details about the frontend side, see this related project (React) or this other related project (Angular)

A REST API built to power a web app for all movie fans
Overview

myFlix was created to serve as a reference in the domain of visual entertainment. Users can create an account and then log into myFlix to have access to information about different movies. They can search for movies, filter results based on different criteria and create lists of favorites.

Purpose

Learn and consolidate my skills with backend technologies during my full-stack web development training. Develop and showcase a user-friendly, appealing and complete web application (client side and server side) built using JavaScript technologies and the MERN tech stack. This page concerns the backend part.

Objective

Build an API from scratch to power myFlix and ensure easy interactions for users whenever they’re accessing the web app to read and save information about different movies, as well as allowing me to put into practice my learnings regarding different backend aspects for a fully functional server-side API.

Tech summary

The server-side of myFlix includes the server, business logic and business layers of the web app. It consists of a well-designed REST API and architected database built using JavaScript, Node.js, Express, and MongoDB. The REST API is accessed via commonly used HTTP methods (GET, PUT, POST and DELETE). Similar methods (CRUD) are used to perform operations on data stored in a non-relational database.

Tools used
  • Node.js and Express framework
  • Node version manager (nvm) and Node package manager (npm)
  • PostgreSQL and MongoDB
  • Heroku
  • Postman

Retrospective


The original objective of this project (along with this frontend React related project and this frontend Angular related project) was to develop the backend architecture and build an API from scratch to feed myFlix, which align with the final result.

Being in my first steps with the creation of an API, the most challenging thing for me was to understand the differences between front-end coding and back-end coding, and get used to work with backend technologies.

A lot of things were new to me, such as Node.js, endpoint creation, non-relational databases and the affiliated tools (Mongoose, Mongosh and MongoDB Atlas) and API deployment process. Learning about web development security measures also demanded a higher level of concentration, but it was, at the end, very rewarding to see my web app being structured in a more secure way after the implementation of various protection methods in my codes.

All these elements and challenges, however, were also what I appreciated the most in this project, because they will allow me to develop other similar projects to which I will be able to add even more functionalities due to the experience I acquired. All the knowledge learned in this work opened me the doors to design effectively and methodologically other projects needing backend logic, since the languages ​​and tools used here (MERN tech stacks) are very common and versatile.

Coding this project gave me the opportunity to acquire a good understanding of how Node.js, Express and MongoDB works both in theory and in practice, thus adequately preparing myself to use the latest technology of the MERN tech stack: React.
